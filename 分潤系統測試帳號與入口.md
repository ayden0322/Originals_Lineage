## 測試帳號

### 主後台（管理者）

| 角色 | 帳號 | 密碼 | 權限 |
|------|------|------|------|
| 模組管理者 | `originals@gmail.com` | `originals123` | 全模組權限（含代理分潤 view/manage）|
| 平台超級管理者 | `admin@admin.com` | `admin123` | 全平台權限 |

→ 登入後左側選單 **代理分潤** 展開可看到 5 個子項：
- 代理管理
- 玩家歸屬
- 結算管理
- 退款沖銷
- 分潤設定

### 代理後台（測試用代理帳號）

| 代理 | 層級 | 帳號 | 密碼 | 比例 | 父代理 | 備註 |
|------|------|------|------|------|--------|------|
| A001 測試A代理 | 一級代理（A）| `agent_a01` | `agent123` | 30% | - | `can_set_sub_rate=true`，可看到「子代理」選單 |
| B001 測試B代理 | 二級代理（B）| `agent_b01` | `agent123` | 60% | A001 | 看不到「子代理」選單 |

### 玩家測試 ID

| 用途 | ID | 備註 |
|------|----|----|
| 測試玩家歸屬 | `11111111-1111-1111-1111-111111111111` | 已綁定到 B001 |

## 分潤計算範例對照

| 情境 | 玩家儲值 | A 拿 | B 拿 | 公式 |
|------|---------|------|------|------|
| 玩家歸屬 B001 | 100 元 | **12 元** | **18 元** | 團隊池 100×30%=30；B 拿 30×60%=18；A 留 30-18=12 |
| 玩家歸屬 A001（A 直推）| 100 元 | **30 元** | - | 100 × 30% |
| 無歸屬（SYSTEM）| 100 元 | 0 | 0 | 進 SYSTEM 虛擬代理（純報表閉環）|

## 常用操作測試清單

- [ ] 主後台登入 → 進「代理管理」看樹狀（A001 + B001）
- [ ] 新增 A002 一級代理
- [ ] 新增 B002 掛在 A002 底下
- [ ] 試試「升格」B001（兩步驟 Modal）
- [ ] 試試「轉組」B 到不同 A
- [ ] 「停權」A001 應該被擋（旗下還有 B001）→ 提示先轉移
- [ ] 玩家歸屬頁查 `11111111-...` 應顯示綁定到 B001
- [ ] 代理 A001 登入 → 看「子代理」頁應該有 B001
- [ ] 代理 A001 試試調整 B001 比例（because `can_set_sub_rate=true`）
- [ ] 代理 B001 登入 → 看不到「子代理」選單
- [ ] 代理產生新推廣連結 → 開 QR Code → 下載 PNG
- [ ] 結算管理選 B001 → 目前還沒有結算紀錄（需等結算日 5 號 cron 跑）

## Docker 相關指令速查

```bash
# 啟/停容器
docker compose up -d
docker compose stop

# 看後端日誌
docker logs -f originals-backend

# 重啟後端（套用程式碼變更通常不用，因為 ts-node-dev 自動 reload）
docker compose restart backend

# 重啟前端（新增頁面後若 404 通常需要 restart）
docker compose restart frontend

# 進 PostgreSQL CLI
docker exec -it originals-postgres psql -U platform_admin -d platform_db

# 看 commission 資料表
docker exec originals-postgres psql -U platform_admin -d platform_db -c "\dt commission_*"

# 看當前代理列表
docker exec originals-postgres psql -U platform_admin -d platform_db -c "SELECT code, name, status FROM commission_agents;"

# 看當前系統設定
docker exec originals-postgres psql -U platform_admin -d platform_db -c "SELECT key, value FROM commission_settings;"
```

## 重要尚未串接的部分

### ⚠️ 儲值流程串接（待辦）
分潤計算引擎 `CommissionEngineService` 監聽 `commission.recharge.paid` 事件，
但目前**沒有任何模組會 emit 這個事件**。

需要在以下時機之一加 emit：
- 金流 callback 確認儲值成功（`PaymentService` 的 `payment.paid` event 之後）
- 或新建獨立的「儲值」模組

範例 emit：
```typescript
this.eventEmitter.emit('commission.recharge.paid', {
  transactionId: tx.id,        // payment_transactions.id
  playerId: order.playerId,    // website_users.id
  amount: tx.amount,
  paidAt: tx.paidAt,
});
```

→ 串接後分潤計算才會自動跑、結算才有資料。
